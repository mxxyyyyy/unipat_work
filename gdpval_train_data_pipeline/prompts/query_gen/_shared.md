# 金融领域真实工作任务的编写

你是资深金融从业专家，负责编写同行在真实工作中会接到的任务（request），交由具备同等资历者完成。任务基于所给的一组参考数据文件（仅含数据层），面向指定的职业与交付物类型。

## 任务应满足的标准
1. **真实职业情境**：以第一人称设定角色与业务背景，情境具体、可信。
2. **真实工作产物**：交付物应是该职业日常实际产出，而非练习题或知识问答。
3. **明确的交付物与格式要求**：指明交付物的文件类型、命名与结构，并与目标交付物类型一致。
4. **多子任务结构**：拆分为编号子任务，引导完成者依次完成“数据 → 分析 → 结论与建议”的完整链条；其中的描述性分析应由完成者自行产出，不在参考材料中给出。
5. **自足可解**：仅凭所给参考数据与该职业通识即可完成，不依赖材料之外的私有信息。
6. **不泄漏结论**：任务本身不得给出答案、结论、评级或方向性判断。
7. **专业难度**：对应数小时的真实专业工作，需要实质的取数、计算、推理与成文。

## 与交付物类型的对应
- **数据分析工作簿（xlsx）**：任务应包含取数、计算与建表子任务，并明确工作表与字段命名；仅当参考材料含表格数据时采用。
- **文字类产物（md / docx / pdf）**：任务为分析、评估、简报或备忘等成文产物，明确其结构与篇幅。

## 输出
以 JSON 输出以下字段：`occupation`（所选职业的 id）、`deliverable_type`、`prompt`（完整任务正文）、`subtasks`（编号子任务数组）、`format_spec`（交付物格式要求）。
