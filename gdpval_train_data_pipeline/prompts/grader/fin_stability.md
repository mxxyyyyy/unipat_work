# 金融稳定与系统性风险：评分核验要点与常见问题识别

## 核验要点（如何快速验证关键计算与口径）
- **信贷 / GDP 缺口**：缺口 = 水平 − 单边 HP 趋势（λ=400000），而非 credit / GDP 本身；单位为百分点。若交付物把"水平"当"缺口"、或用双边 HP → 客观项 0 分。
- **DSR**：核验 `i·D/[1−(1+i)^(−s)]/Y` 的分子分母口径（利率、债务存量、剩余期限、收入频率一致）；仅用"利息 / 收入"而漏掉摊还也可接受，但口径须自洽。
- **比率**：CET1 = CET1 / RWA、杠杆率 = Tier1 / 总敞口、LCR = HQLA / 30 天净流出、NSFR = ASF / RSF；抽取 1–2 个用参考材料反算校对，并判断阈值（LCR / NSFR ≥ 100%）是否正确。
- **CCyB 映射**：缺口 ≤ 2 → 0、≥ 10 → 2.5%、间线性——据此复算建议档位；档位跳错即 0 分。
- **压力测试**：CET1 谷值 = 起始 − 峰谷降幅；资本缺口 = 最低要求 − 谷值（为负才有缺口）；抽查 PPNR、损失率 × 敞口的乘算与 RWA 迁移方向。
- **估值分位与偏离**：分位 / Z-score 应基于参考材料给定的样本；若样本仅有近数年却报出更长历史区间的分位 → 属编造，accuracy 扣分。
- **单位与量纲**：% of GDP / % of RWA / bp / 倍数是否混用；名义与实际、同比与环比是否标错。
- **判断一致性**：脆弱性刻度（low / moderate / elevated / high）、CCyB 上调 / 维持 / 释放、是否通过等结论方向，与算出的数值及 answer_key 方向是否一致（接受合理变体，只看方向与依据）。

## 常见问题与识别
- **只搬运数据而不计算**：满页复述参考序列，却无缺口、分位、谷值或档位 → 对应客观项与"完成计算"必过项给 0 分。
- **编造数字**：报出参考材料中无法复算的 CET1 谷值、资本缺口或历史分位 → accuracy 与该客观项清零；反算对不上即为编造。
- **口径混用**：把缺口当水平、DSR 当利率、杠杆率当 CET1、% of RWA 当 % of GDP —— 逐口径核验，错即 0 分。
- **方向性表述空转**："风险上升、需密切关注"却无量化依据、无支柱定位、无传导机制 → 判断项按空泛无据扣分。
- **声称交付却未交付或仅交半成品**：正文说"见附表 / 见 xlsx"但文件缺该工作表或列、章节缺支柱、pdf 说要图却无图 → 三大失败模式必过项（`required`）判 0 分：①交付物文件是否非空、可打开；②是否确用参考材料关键数据；③文件类型、名称、工作表是否符合 `format_spec`。
- **伪压力测试**：给出结果却无情景路径与起始横截面来源、损失凭空得出 → 视为编造，相关项清零。
