# 银行稳健性任务的客观核验点、判断要素与守门条目

## 客观可核验点清单（objective 项主干，应尽量充分，可从参考材料或公式直接核验）
**口径与公式（数值正确性可复算）**：
- CET1 ratio = CET1 / RWA；Tier1、Total capital ratio；leverage ratio = Tier1 / 敞口；RWA density = RWA / 总资产。
- NPL/NPE ratio = 不良 / 总贷款；**coverage = 拨备 / 不良**（而非 / 总贷款）；cost of risk = 减值 / 总贷款（年化 bps）；NCO rate = 净核销 / 平均贷款；美式 reserve coverage = ALLL/ACL / noncurrent；Texas ratio =（不良 + OREO）/（有形普通股权益 + 损失准备）。
- ROE = 净利润 / 平均权益；ROA = 净利润 / 平均资产；**NIM = 净利息收入 / 平均生息资产**；cost/income = 营业费用 / 营业收入；RoRWA = 净利润 / RWA。
- LCR = HQLA / 30 天净流出；NSFR = ASF / RSF；LTD = 贷款 / 存款。
- 变动量：序列的 bps 或百分点变动、同比或环比、峰谷 depletion（压测起点 CET1 − 不利情景谷值 CET1）。
**对标结构（可核验）**：加权平均与简单平均口径正确（EBA 报告采用加权平均）；四分位、分位与 5–95 区间；目标体系或机构的分位位置标注正确。
**结构与格式**：xlsx 点名的工作表（如 `Data`、`Ranking`、`Summary`）均存在；行数 N 等于参考面板行数；单位一致（% 与 bps 与百万本币）；四支柱或 CAMELS 章节齐备。

示例：`[+2] coverage 按 拨备/不良 计算，且各国数值与参考材料一致`；`[+2] NPL 8 季变动以 bps 表示且计算正确`；`[+1] 工作表 'Ranking' 存在且含分位列`；`[+2] 加权平均按敞口或资产加权（非简单平均）`。

## judgment 项应覆盖的分析要素（以 answer_key 作软性参照，接受合理变体）
- **资本**：是否正确判断资本水平与趋势，以及 **buffer to MDA 与压力后 CET1** 的含义（哪一体系距缓冲或 MDA 更近、哪一体系 depletion 更大）。
- **资产质量**：是否识别出不良恶化或改善的主要体系及其**驱动因素**，是否以 Stage 2 占比与 cost of risk 作**前瞻**判断（而非只看静态 NPL），是否判断 coverage 的充足性。
- **盈利**：是否区分**盈利质量与可持续性**（NIM 驱动与一次性因素、拨备回拨支撑的 ROE），ROE 是否与资本成本或 COE 相较。
- **流动性与融资**：是否解读存款结构（未保险与无息占比）、LTD 与 LCR/NSFR 余量，识别融资脆弱性。
- **敏感性与综合**：是否指出 AFS/HTM 未实现损失对资本的隐性侵蚀与利率敏感性；综合判断是否**内在一致**（各支柱结论不相互矛盾）且与参考材料的数据方向吻合。

judgment 项每条应在 `anchor_ref` 引用 answer_key 的支撑片段；判分以方向与要点是否到位为准，不要求与 answer_key 逐字一致。

## 应据以设立守门与负向条目的常见问题
- **只列数字不作解读**：将指标搬入表格却无对标结论、无驱动分析、无前瞻——judgment 项与 structure_style_relevance 扣分。
- **coverage、MDA 或 depletion 口径错误**（拨备除以总贷款、将当期 CET1 当作压力后）——accuracy 客观项判 0。
- **以简单平均冒充加权平均**、分位算错或分位位置标反——对标客观项判 0。
- **编造参考材料所无的数据**（外部行业均值、逐笔敞口、目标价、官方 CAMELS/SREP 评分）——accuracy 守门项判 0（对应「忽略或编造参考数据」这一失败模式）。
- **静态看待不良**：只报当期 NPL，不以 Stage 2、CoR 或迁徙作前瞻——前瞻判断项不给分。
- **承诺却未交付**：正文称「见附表」「见 Summary」而文件缺该工作表或章节，或结论与所给数据方向相反——required 守门项判 0。

## required 守门项（针对三大失败模式，各至少一条）
- 交付物文件确实产出、非空、可打开，且点名的工作表与章节齐全。
- 关键指标确以参考材料数据按标准银行口径计算，未编造外部数字。
- 交付物格式（文件类型、命名、工作表、篇幅，及四支柱或 CAMELS 结构）与 `format_spec` 一致。
