# 金融稳定与系统性风险：评分细则的领域要点

## 客观可核验点（细则主干，可从参考材料或格式直接核验，无需判断）
按任务涉及的指标，把口径与公式写成可复算的客观项：
- **信贷 / GDP 缺口**：`gap = 100·(credit/GDP) − trend`，trend = 单边 HP 滤波 λ=400000；核验缺口数值与 Z-score 计算是否正确。
- **偿债比率 DSR**：`DSR = i·D / [1−(1+i)^(−s)] / Y`（i = 季度利率、D = 债务存量、s = 剩余期限季数、Y = 季度收入）；核验分子分母口径。
- **CET1 比率** = CET1 / RWA；**一级杠杆率** = Tier1 / 总敞口；**LCR** = HQLA / 30 天净流出（≥ 100%）；**NSFR** = ASF / RSF（≥ 100%）——核验比率计算与阈值对照。
- **估值分位**：某指标当前值相对样本分布的百分位 / Z-score；**偏离** = 当前值 − 历史均值（或中位数），单位与方向正确。
- **CCyB 缓冲指引映射**：缺口 ≤ 2 → 0；≥ 10 → 2.5% RWA；间线性——核验档位映射数值。
- **压力测试**：CET1 谷值 = 起始 CET1 − 峰谷 CET1 降幅；资本缺口 = 监管最低要求 − 谷值 CET1（若为负）；PPNR、损失率 × 敞口的乘算。
- **结构类**：交付物文件名与类型与 `format_spec` 一致；FSR 章节含"逐支柱脆弱性 + 总体判断"；xlsx 点名工作表（如 `Indicators` / `Guide`）与列存在；每个子任务各对应至少一条客观项。
- **单位与量纲**：% of GDP / % of RWA / bp / 倍数不得混用；同比与环比、名义与实际标注正确。

## 判断项应覆盖的分析要素（以 answer_key 软支撑，接受合理变体）
- **总体脆弱性定性刻度**方向合理（low / moderate / elevated / high），且与算出的分位、偏离一致。
- **正确识别最被拉伸或最脆弱的支柱与资产类**（如估值 vs 杠杆 vs 融资错配），并有数据支撑。
- **传导与放大机制判断**得当：火线抛售、保证金螺旋、久期错配导致挤兑、主权-银行链条、NBFI 杠杆去化、互联性。
- **政策与仓位含义方向合理**：CCyB 上调、维持或释放的理由；借款人工具是否收紧；对冲或减仓方向。
- **相机判断**：说明为何偏离机械指引（信贷动能、房价、结构性因素），而非仅照抄指标读数。
- 每条判断项在 `anchor_ref` 引用 answer_key 的支撑片段；只要方向正确、有参考材料支撑即给分，允许与 answer_key 表述不同。

## 常见的敷衍与流于形式手法（据此设置守门与必过条目）
- **只复述参考数值而不做计算或判断**（无缺口、无分位、无谷值）→ 设"确已完成要求的计算并给出脆弱性判断"必过项。
- **编造具体数字**：凭空报出 CET1 谷值、资本缺口或分位，而非从参考材料算出 → 设 accuracy 必过项"关键数值可由参考材料复算"。
- **方向性表述空转**：满篇"风险上升需警惕"却无量化依据、无支柱定位 → 判断项须要求数据支撑。
- **口径混用**：把信贷 / GDP 缺口当成 credit / GDP 水平、把 DSR 当利率、把杠杆率当 CET1 → 客观项按口径严格核验。
- **声称交付却未交付或仅交半成品**：说要出 xlsx 却给 md、工作表或列缺失、章节缺支柱 → 对应三大失败模式设必过项（`required: true`）：①确已产出要求的交付物文件（非空、可打开）；②分析确用参考材料关键数据而非编造；③格式、文件名、工作表与 `format_spec` 一致。
