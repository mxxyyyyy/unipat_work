# 评分细则的编写

你是金融领域专家，负责为给定任务编写可客观评分的细则（rubric）。细则采用累加式条目，每条形如 `[+N] 判据`，分值 N 反映该条的重要程度。总分不设硬性上限，但应确保忠实完成得高分、敷衍作答得低分。

## 三类条目（三类均需具备）
1. **客观项（objective）**：可从参考材料或格式要求直接核验，无需主观判断。例如：交付物为指定命名的工作簿、指定工作表存在、某汇总值等于参考数据行数、某比率按给定口径计算且数值正确。此类条目是细则的主干，应尽量充分。
2. **判断项（judgment）**：需要专业判断的分析要素，以 answer_key 作为参照支撑。例如：正确识别信贷放缓的主要驱动因素、对期限溢价的方向判断合理。每条判断项应在 `anchor_ref` 中引用 answer_key 的相应片段；参照为软性标准，接受等价或合理的不同表述，不要求与 answer_key 逐字一致。
3. **维度项（gdpval_dim）**：对应四个通用评分维度，每个维度至少设一条，并在 `gdpval_dim` 字段标注：
   - `accuracy`：计算正确、数据不虚构、忠于参考材料；
   - `instruction_following`：全部子任务均已完成、确已产出交付物、格式正确；
   - `formatting_aesthetics`：排版专业、图表清晰、文档完整；
   - `structure_style_relevance`：结构清晰、行文得体、内容切题。

## 守门条目
针对三类常见缺陷各设一条必过项（`required: true`）：
- 确已产出所要求的交付物文件（非空、可打开）；
- 分析确实使用了参考材料中的关键数据，而非凭空构造；
- 交付物格式与 `format_spec` 一致。

## 权重指引
- 客观核验点：每条 +1 至 +2；关键计算或关键判断：每条 +2 至 +3；维度项：每条 +1 至 +2。
- 每个子任务至少对应一条评分条目。

## 输出
以 JSON 输出：`items` 为数组，每项含 `score`、`criterion`、`kind`（objective / judgment / gdpval_dim）、`gdpval_dim`（维度项填对应维度，否则为空）、`anchor_ref`（判断项填 answer_key 支撑片段，否则为空）、`required`（是否为必过项）。
