# 金融稳定与系统性风险：任务质量审查的领域要点

## 贴近实务（representative）的判准与反例
**贴近真实产物的特征**：以央行、监管或买方风控的口吻，围绕 Fed FSR 四支柱（估值 / 借贷 / 杠杆 / 融资）或 ECB FSR、BIS 稳定章节的真实结构展开；用词准确（信贷 / GDP 缺口、DSR、CET1、LCR、CCyB、EBP、CISS）；有明确决策受众（FSR 章节读者、审慎决策委员会、风控或交易台）；要求完成 取数 → 计算 → 判断 → 政策 / 仓位含义 的闭环。
- **正例特征**：要求给出脆弱性定性刻度（low / moderate / elevated / high）并说明依据；要求把缓冲指引指标映射到 CCyB 档位；要求算压力后 CET1 谷值与资本缺口；要求识别可挤兑负债与久期错配的传导。
- **反例（判 representative = 0）**：
  - 习题化：仅"请解释什么是系统性风险""比较 CET1 与杠杆率的定义"——无交付物、非数据驱动。
  - 口径与受众错位：让金融稳定分析师写个股买入评级、写货币政策利率决议（分属权益研究与货币政策领域）。
  - 杜撰情境："假设某虚构银行有 100 元资本"——非真实数据、非真实产物。

## 可解性（solvable）的常见陷阱
以下"结论"实则需要参考材料之外的数据或信息：
- **依赖当期政策变量**：最新一期实际 CCyB / SyRB 档位、当前政策利率、G-SIB 名单与附加档位——参考材料未含却要求据此判断，则 solvable = 0。
- **依赖机构级非公开明细**：单家银行的组合级 PD / LGD、逐笔敞口、内部模型 RWA——汇总序列无法反推，自下而上压力测试因此不可解。
- **依赖外部基准分位**：要求"相对某长历史区间的分位"，但参考材料仅给近数年样本，无法计算长历史分位。
- **依赖跨库数据**：FSB 全球 NBFI 规模、IMF GFSR 特定图表、他国对照数据——参考材料未含则不可解。
- **口径必需但缺失**：算信贷 / GDP 缺口需明确 HP 的 λ 与单 / 双边；算 DSR 需利率、债务存量、剩余期限；缺关键口径且无法从参考材料合理默认时，视 answer_key 是否仍可得，必要时 solvable = 0。
- **反向核对**：将 answer_key 的关键结论（脆弱性刻度、建议档位、CET1 谷值、是否通过）逐条追问——每一步是否都能仅凭参考数据与领域通识推出？只要有一步依赖参考材料中根本没有的数据，即 solvable = 0。

## 常见不合格模式清单
1. **泄漏（zero_leakage = 0）**：任务或参考材料中已写出方向性判断（"估值 elevated""风险偏高""建议上调缓冲""该行资本充足 / 不足"）、给出结论性分位或谷值。
2. **不可解（solvable = 0）**：见上——依赖当期政策档位、机构级明细、长历史基准或跨库数据。
3. **无交付物或格式含糊（has_deliverable = 0）**：仅说"分析一下融资风险"，不点明产物类型、文件名、结构、篇幅或工作表与列。
4. **难度不足（non_trivial = 0）**：仅查一个 CET1 读数、仅复述一条序列、仅做单指标同比——非数小时专业工作。
5. **歧义（unambiguous = 0）**：涉及缺口、DSR、比率却不点明口径；情景或起始状况不全却要求压力测试结果。
6. **拼凑（coherent = 0）**：把估值、跨境敞口、个股估值、宏观利率预测硬塞进同一题，不围绕单一交付物层层递进。
7. **领域错配（on_domain = 0）**：偏成货币政策利率路径、主权评级或纯银行个体信用研报（属其他领域），而非系统性 / 宏观审慎视角。
